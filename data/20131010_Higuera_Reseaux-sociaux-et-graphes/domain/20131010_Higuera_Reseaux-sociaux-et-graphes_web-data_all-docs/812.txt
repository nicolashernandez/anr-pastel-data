PJE Analyse de comportement avec Twitter

Le but de ce TP est :
• d'ajouter une visualisation des tweets sous forme de graphe,
• d'apporter des informations globales sur l'ensemble des tweets par l'analyse du graphe.

Représentation sous forme de graphe

Un graphe de tweets peut être déni de la manière suivante. Les noeuds du graphes sont les tweets répon-
dants à une requète, une arête entre deux tweets existe lorsque la distance est inférieure à un certain seuil.
La distance peut être celle utilisée dans les derniers TP (nombre de mots en commun).

Plusieurs possibilités pour dessiner ce graphe :
• Utiliser une application externe comme graphviz avec une communication du graphe par chier au

format dot,

• Utilier une API de graphe en java comme JgraphT, JUNG, ou Gephi,
• Dénir les classes permettant de dénir un graphe, puis dessiner ce graphe tout d'abord spacialisé

de manière aléatoire, puis spacialisé en utilisant le principe des ressorts (spring layout).

Mesure globable sur le graphe

Le graphe des tweets apporte des informations supplémentaires à la classication. La distance moyenne
entre les tweets peut informer sur l'étendue des opinions sur un sujet ; le degré moyen sur la proximité
des opinions ; le coecient d'agglomération moyen sur la présence de communauté d'opinion sur un sujet.

• Ajouter le calcul de ces mesures dans l'interface.
